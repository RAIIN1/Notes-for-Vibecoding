## 1 如何把模糊的需求变成能用的产品
 STEP1：需求的具体化
 
 需求：用户真正想要的东西、遇到的麻烦、想解决的问题；往往模糊且难以直接落地
 
 业务：用户每天实际在做的事情、他们的工作方式；展现出用户的工作流，从而将需求具体化为一个个详细诉求
 
 STEP2：建立优先级
 
 最优先：最痛、最急、最能解决
 
 先做出最小可用的版本，再慢慢迭代
## 2 基于AI IDE产出原型
#### 2.1 产出前：提示词常用方式
1. 直接口述
	注意不要直接输入纯口述的提示词，需要AI辅助扩写打磨，是一个反复修改确认的过程
2. 导入业务文档
	可以以“帮我参考业务逻辑实现一个单页面应用，用来验证核心玩法功能“开头
3. 让 AI 帮写提示词的提示词
	也是个人平时使用最多的方式，但是一直苦于生成内容有时脱缰，所以重点记录一下教程中介绍的一些限制要求：
	1. 提示词要包含清晰的页面布局描述
	2. 明确数据结构和交互逻辑
	3. 指定技术栈（如 React + Tailwind）
	4. 列出需要实现的核心功能点
	5. ······
#### 2.2 产出中：报错处理
这里重点记录消化一下教程介绍的多次修改无果的应对方式：
- 版本回退至一个可运行的版本重新开始
- 更换模型或调整提示词，将现象、错误信息讲得更具体
- 将“当前代码 + 错误日志 + 预期行为”打包，一次性发给 AI，让它整体重构问题部分
在之前的个人实操中确实经常遇到过多次修改无果的问题，基于对本教程的理解，在此时最重要的应该是避免和AI进入无限的兔子洞，要么改变位置（当前or之前版本），要么改变工具（模型/提示词），要么改变视角（局部or全局）
#### 2.3 产出中进阶：单页面至多页面
这里重点关注页面交互是否正确
#### 2.4 产出后：测试和核对业务清单
对原型测试：在原型上亲自体验一遍全流程，看是否满足业务文档中的诉求，并将无法运行和缺失的部分交给AI修复，之后循环验证。

个人认为这里的一大痛点是测试时新建数据往往耗费较多时间，教程提供了一种很好的解决方式：“测试数据”。

具体而言，让 AI 在界面上生成可以测试的快速数据入口（如“一键填充测试数据”），从而可以直接在界面上点击来验证全流程。

这种方式往往用来检验结果是否符合预期，如果需要检验过程，也可以通过提示词要求AI同时模拟出真实的生成过程。
- 也可以让AI辅助生成一个checklist

## 作业：复刻电商素材工作台

1. 业务梳理提示词
```
角色设定
你是一位资深的前端架构师和 UI 设计师。请帮我从零构建一个名为 **“电商智能素材工作台” (E-Shop Studio)** 的 React 应用原型。

# 核心业务逻辑
这是一个 B2B 电商运营工具，主要解决两个问题：
1.  **批量生产**：输入商品信息，模拟生成“主图+标题+文案”。
2.  **资产复用**：将生成的优质方案“存为模板”，方便下次直接套用。

# 技术栈要求
- **框架**: React 18 + Vite (TypeScript)
- **样式**: Tailwind CSS (追求现代、简洁的 SaaS 风格)
- **图标**: Lucide React
- **状态管理**: Zustand (用于管理商品数据和模板库状态)
- **组件库**: 使用原生 HTML/Tailwind 封装基础组件（Card, Button, Input, Modal），保持代码轻量。

# 数据结构定义 (TypeScript)
请在 `types.ts` 中定义核心类型：
1.  **ProductInfo** (输入信息):
    - `id`, `name` (商品名), `brand` (品牌), `category` (类目)
    - `baseImage` (上传的白底图 URL)
2.  **GeneratedAsset** (生成的素材):
    - `id`
    - `mainImage` (合成后的主图 URL)
    - `title` (生成的 SEO 标题)
    - `copywriting` (1-2句卖点文案)
    - `status`: 'draft' (草稿) | 'saved' (已保存)
3.  **Template** (模板):
    - `id`, `name` (如“大促红色风”), `tags` (如“男装”)
    - `stylePreview` (模板缩略图)

# 页面布局与功能详情

应用采用 **左侧侧边栏 + 右侧主内容** 的布局。

## 1. 侧边栏 (Sidebar)
包含两个导航项：
- **工作台**: 核心生产页面
- **模板库**: 管理已保存的模板

## 2. 核心页面：工作台 (Workbench)
此页面采用 **左右分栏布局**：

### 左侧：配置面板 (Input Panel)
- **顶部**: 切换 Tab ["单品录入", "Excel 批量导入"]。
- **表单**:
  - 商品基础信息输入框（名称、品牌、核心卖点）。
  - 图片上传区域（做一个模拟的点击上传 UI）。
  - **模板选择器 (关键功能)**: 下拉菜单，选项包括“智能默认生成”以及**从 Zustand Store 中读取的已保存模板**。
- **底部**: 一个醒目的“开始生成”按钮。

### 右侧：结果预览区 (Results)
- **空状态**: 提示“请在左侧输入信息”。
- **生成中**: 点击生成后，显示 1.5 秒的骨架屏 (Skeleton) Loading 效果。
- **结果列表**: 以卡片形式展示生成的 1-3 个方案。
- **单个结果卡片功能**:
  - 展示图片、标题、文案。
  - **[编辑]**: 点击弹出弹窗，修改标题和文案。
  - **[存为模板] (核心闭环)**: 
    - 点击后弹出 Modal。
    - 输入“模板名称”和“标签”。
    - 保存后，该方案被存入模板库，且**左侧的模板选择器里能立即选到它**。

## 3. 辅助页面：模板库 (Library)
- 网格展示所有已保存的模板卡片。
- 每个卡片包含“删除”和“立即使用”按钮（点击跳转回工作台并选中该模板）。

# 关键交互逻辑 (Mock Data)
1.  **模拟生成**: 点击“生成”时，不要调用真实 API。请根据输入的商品名，前端**伪造** 3 个结果。
    - 图片使用 `https://placehold.co/400x400?text=Product` 或类似的占位图服务。
    - 标题和文案请根据输入的商品名动态拼接（例如："【2024新款】[输入的名字]...").
2.  **数据持久化**: 使用 Zustand 存储模板数据，初始化时预置 2 个默认模板（如“极简风模板”、“大促风模板”），确保一开始有数据可展示。
```
